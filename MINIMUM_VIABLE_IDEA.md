# Minimum Viable Idea

## Fastest Artifact

Current artifact:

- Vite + Three.js browser prototype.
- Purpose: quickly test sacred circle readability and temple risk/reward.
- Status: useful, but disposable.

This artifact is not the first playable under GamesOS production standards.

## Question It Answers

Can the player understand this loop quickly?

```text
Light is safety -> enemies threaten it -> fighting expands it -> temple risk creates power.
```

## What It Must Not Become

The browser prototype must not become:

- The production architecture.
- The vertical slice.
- The art quality target.
- A reason to avoid UE5.

## Minimum Evidence

The sketch is useful if it produces:

- One 10-second capture showing movement, strike, enemy burst, and circle response.
- One 30-second capture showing temple risk and hammer power.
- Notes on whether the circle and temple read without explanation.

## Next Minimum Production Artifact

UE5 first playable shell:

- One arena map.
- One third-person pawn.
- Sacred circle actor.
- One shadow imp.
- One temple actor.
- Basic combat interaction.
- Play-In-Editor screenshot and clip.

## Timebox

Do not spend more time polishing the browser prototype until:

- UE5 install is blocked by external setup, and
- the work directly informs the UE5 implementation.

Allowed browser work:

- Capture proof.
- Tune first 10-second logic.
- Explore camera framing.

Disallowed browser work:

- Progression systems.
- More weapons.
- Final art/audio.
- Long-term save/settings architecture.
